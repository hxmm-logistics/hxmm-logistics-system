# 真机扫码试运行检查清单

目标：验证物流员工能在真实手机环境完成扫码和状态更新。

## 测试设备

- Android Chrome
- iPhone Safari
- Telegram WebView

## 前置条件

- 手机和服务器在同一网络，或使用可访问的公网 HTTPS 地址。
- 摄像头扫码必须使用 HTTPS，`localhost` 只适合电脑本机测试。
- 测试单号使用平台单号 `platform_tracking_no`，例如 `HX202605210001`。

## 测试步骤

1. 打开扫码页。
2. 确认浏览器弹出摄像头权限。
3. 允许摄像头权限。
4. 扫描包含平台单号的二维码。
5. 确认输入框自动填入平台单号。
6. 依次点击：
   - 已到木姐
   - 已到曼德勒
   - 派送中
   - 已签收
7. 查询该平台单号，确认时间轴新增对应节点。

## 记录项

- 摄像头是否能打开。
- 首次扫码耗时。
- 弱光环境是否可识别。
- Telegram WebView 是否阻止摄像头。
- 页面是否横向溢出。
- 按钮是否容易误触。
- 重复点击已签收是否重复新增事件。

## 通过标准

- Android Chrome 和 iPhone Safari 均可扫码。
- Telegram WebView 如不可扫码，必须能手动输入平台单号继续操作。
- 已签收重复点击不新增重复事件。
- 状态不能从已签收倒退到派送中。
